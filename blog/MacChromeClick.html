Mac Trackpad javascript click events

Today I learned that the Macbook touchpad in Google Chrome handles javascript click events not intuitively. This seemingly edge case scenario became a weird find for me.

This scenario:

  • Macbook Pro OSX 10.8.5
  • Google Chrome 30.0.1599.101
  • jQuery 1.7.2 +
  • an HTML select box

HTML:

<select id="select"> <option value="1">1</option> <option value="2">2</option> <option value="3">3</option> <option value="4">4</option> <option value="5">5</option> </select>

Javascript:

$(document).on('ready', function(){ var $select = $('#select'); console.log($select); console.log($select.val()); $select.click(function(){ console.log('click'); console.log($select.val()); }); $select.change(function(){ console.log('change'); console.log($select.val()); }); $select.on('click', function(){ console.log('onclick'); console.log($select.val()); }); $select.on('change', function(){ console.log('onchange'); console.log($select.val()); }); });

I noticed that the click event doesn't fire when either lightly tapping or "hammer clicking" on the trackpad. The correct solution for our issue was to handle the change event and not the click event.

This only seems to happen in Chrome.
